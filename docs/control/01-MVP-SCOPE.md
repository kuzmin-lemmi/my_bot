# MVP SCOPE — границы MVP v1

## Что входит в MVP v1

### Ввод целей
- Добавление целей через Telegram
- Добавление целей через мобильное приложение
- По умолчанию цель без даты ставится на "сегодня"
- Поддержка нескольких целей одним сообщением (каждая строка = отдельная цель)

### Данные цели
- Заголовок (обязательный)
- Заметка / комментарий (опционально)
- Дата (обязательно)
- Время (опционально, но поддерживается)
- Приоритет (поле есть, логика пока единая)

### Мобильное приложение (Android, Flutter)
- Экран "Сегодня"
- Календарь (месяц) для выбора даты и просмотра/добавления целей на дату
- Карточки целей с действиями
- Настройки напоминаний
- Базовая офлайн-устойчивость (желательна, не full offline-first)

### Действия по цели
- Выполнено (с подтверждением)
- Отложить (варианты времени)
- Перенести на завтра
- Отменить

### Напоминания
- Повторные уведомления по интервалу (реалистично под Android)
- Звук (в мягком режиме — реже/тише по настройкам)
- Напоминания по очереди (чередование целей)
- Окно напоминаний (по умолчанию 09:00–21:00)
- Тихий период (например 14:00–17:00, настраиваемый)
- Режимы настойчивости (минимум 3)
- Эскалация частоты при игнорировании (настраиваемая/простая v1)
- Глобальная пауза всех напоминаний на X минут

### Логика дня
- Если цель не выполнена к концу дня — автоматически переносится на завтра
- Утром приложение спрашивает/показывает подтверждение списка перенесённых (soft confirmation)

### Журнал
- Журнал действий по целям (минимум события create/complete/snooze/move/cancel/auto-move)

## Что НЕ входит в MVP v1
- Мультипользовательский режим
- Совместная работа
- Веб-версия
- Интеграция с Google Calendar
- AI-функции
- Сложная аналитика
- Теги/проекты/категории (как логика)
- Глубокая кастомизация дизайна
- iOS

## Минимальная демо-версия (Definition of "уже можно пользоваться")
- Цель можно добавить (Telegram/телефон)
- Цель видна в приложении на нужную дату
- Напоминания приходят по интервалу
- Можно отметить "выполнено"/"отложить"/"завтра"/"отменить"
- Напоминания прекращаются/меняются корректно после действий
