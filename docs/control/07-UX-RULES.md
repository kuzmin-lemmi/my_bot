# UX RULES — правила интерфейса (обязательны для UI-модели)

## Главная идея UX
Это не "менеджер проектов". Это инструмент фокуса и дожима по важным делам.

## Обязательные UX-правила
1. Быстрое действие важнее красоты
2. Главный экран отвечает на 3 вопроса:
   - Что у меня сегодня?
   - Что сейчас требует внимания?
   - Что нажать?
3. Подтверждение использовать точечно
4. UI не придумывает статусы
5. UI не меняет смысл кнопок
6. Порядок целей помогает действию
7. Даже hard-режим не должен лишать пользователя контроля (snooze/pause/cancel)

## Рекомендуемый порядок на "Сегодня"
- active (по времени, затем без времени)
- snoozed
- completed/canceled (внизу или по toggle)

## Анти-паттерны (запрещено в MVP)
- перегруженные фильтры
- много вкладок
- тяжёлые анимации
- AI-советы вместо скорости
- сложный онбординг

## Тон интерфейса
- короткие понятные тексты
- без канцелярита
- без мотивационных баннеров
