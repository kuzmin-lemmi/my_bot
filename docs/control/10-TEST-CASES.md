# TEST CASES — обязательные сценарии проверки MVP

## A. Telegram (одна цель)
1. Отправить одну цель без даты
2. Проверить target_date = today
3. Проверить появление в "Сегодня"

## B. Telegram batch (несколько строк)
1. Отправить 3 строки
2. Проверить создание 3 целей

## C. Создание через телефон на выбранную дату
1. Выбрать дату в календаре
2. Создать цель
3. Проверить target_date

## D. Выполнено
1. Нажать "Выполнено"
2. Подтвердить
3. Проверить status=completed, completed_at
4. Проверить остановку напоминаний

## E. Snooze
1. Отложить на 30 мин
2. Проверить status=snoozed, snooze_until
3. После истечения — active

## F. Move to tomorrow
1. Нажать "Завтра"
2. Проверить target_date +1 день и событие moved_to_tomorrow

## G. Cancel
1. Нажать "Отменить"
2. Подтвердить
3. Проверить status=canceled и остановку напоминаний

## H. Quiet period
1. Настроить 14:00–17:00
2. Проверить отсутствие уведомлений в период

## I. Global pause
1. Пауза на 30 минут
2. Проверить отсутствие уведомлений и восстановление после

## J. Эскалация
1. Включить escalation
2. Игнорировать 2–3 уведомления
3. Проверить уменьшение effective interval (не ниже лимита)

## K. Автоперенос
1. Есть active/snoozed цели на сегодня
2. Выполнить daily rollover
3. Проверить перенос и событие auto_moved_to_tomorrow

## L. Оффлайн (желательно)
1. Открыть без сети
2. UI не падает
3. После сети sync работает
