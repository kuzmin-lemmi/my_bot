# DEFINITION OF DONE — критерии готовности задач и MVP

## Для UI-задач (Gemini)
Готово, если:
- соответствует 06-UI-SPEC.md
- не нарушает 04-API-CONTRACT.md
- не придумывает новые статусы/смыслы кнопок
- есть ручная проверка
- есть HANDOFF

## Для LOGIC-задач (Claude)
Готово, если:
- соответствует 03-DOMAIN-MODEL.md и 05-STATE-MACHINE.md
- API описан/обновлён в 04-API-CONTRACT.md
- есть базовые валидации/ошибки
- есть шаги проверки/тесты
- есть HANDOFF

## Для интеграции
- UI и backend на одном контракте
- нет "временных" полей без фиксации
- проходит нужный сценарий из 10-TEST-CASES.md
- есть handoff (при необходимости от обеих ролей)

## MVP готов к личному использованию, если:
1. Цели можно вносить из Telegram и телефона
2. Есть календарь месяца и выбор даты
3. Есть экран "Сегодня" с действиями
4. Повторные напоминания работают
5. Есть тихий период и глобальная пауза
6. Напоминания корректно прекращаются после действий
7. Автоперенос на завтра работает
8. Журнал действий фиксируется
9. Приложение можно реально использовать несколько дней
